# 互动小说系统：HTML/CSS/JS 分离设计指南

本文档详细阐述自动化小说创作系统中**互动小说**的核心设计理念：将页面结构（HTML）、视觉样式（CSS）、交互逻辑（JS）与最终渲染（互动师）完全分离。这种架构使 AI 能够专注于剧情相关的结构生成和风格定制，而交互行为由预置的高质量控件库提供，最终由互动师统一交付，实现高效、灵活且稳定的互动小说创作。

---

## 1. 设计背景与核心思想

在传统的互动小说生成中，AI 往往需要一次性输出包含样式、脚本和结构的完整 HTML，导致：

- **样式与逻辑耦合**：修改样式可能破坏交互功能，反之亦然。
- **效率低下**：每次生成都需要重复编写相同的基础控件代码。
- **视觉风格单一**：样式难以灵活变化，难以匹配多变的剧情氛围。
- **交互体验不稳定**：控件行为可能因 AI 输出差异而表现不一致。

为解决这些问题，我们引入**职责分离**架构，将页面拆解为四个独立组件，每个组件由不同的智能体（Agent）或系统模块负责：

| 组件 | 角色 | 核心职责 |
|------|------|----------|
| **HTML框架师** | 结构生成 | 生成语义化的 HTML 骨架，包含剧情文本、交互控件容器和控件库引用，不包含任何样式或内联脚本。 |
| **CSS样式师** | 视觉定制 | 生成独立的样式代码（`<style>` 标签），使用约定的类名为所有控件提供视觉表现，可自由设计以匹配剧情氛围。 |
| **JS控件库** | 交互实现 | 预置的 JavaScript 模块，负责扫描页面中的控件容器，动态创建 UI 并绑定事件，返回用户操作结果，可包含动画和效果。 |
| **互动师** | 交付渲染 | 拼接样式与骨架，解析资源引用，渲染交互界面，等待并返回用户操作结果。 |

通过这种分离，AI 只需生成结构（HTML框架师）和样式（CSS样式师），交互行为由成熟稳定的控件库提供，最终由互动师统一整合。这样不仅提升了生成效率，还保证了视觉的灵活性和交互的可靠性。

---

## 2. 组件职责详解

### 2.1 HTML框架师（Structure Generator）

**职责**：根据场景描述（如剧情文本、交互需求）生成纯 HTML 片段，不含任何样式或内联脚本。该片段必须遵循以下规范：

- **根容器**：必须提供一个唯一的根元素（如 `id="scene-{{timestamp}}"`），作为样式和脚本的作用域。
- **内容区域**：按语义划分视觉描述、文本段落等，可使用自定义数据属性（如 `data-type`）标识，便于后续增强。
- **控件容器**：每个交互控件必须是一个独立的容器，包含两个自定义属性：
  - `data-control-type`：标识控件类型（如 `"choice"`、`"dice"`），由开发者定义。
  - `data-config`：存储控件配置的 JSON 字符串，内容格式由控件类型决定。
- **资源引用**：所有外部资源（图片、音频、JS库）使用统一协议（如 `src="id:资源ID"`）引用，资源 ID 由系统管理。
- **库引用**：必须包含对 JS 控件库的引用（如 `<script src="id:控件库ID">`），确保交互逻辑被加载。
- **禁止**：不得包含任何内联样式或额外脚本（除库引用外）。

**示例输出**：
```html
<div id="scene-1680000000000" class="scene-container">
  <div class="text-layer">
    <div class="text-box">
      <p class="narrative-text" data-fulltext="你站在一扇古老的木门前。">你站在一扇古老的木门前。</p>
      <div class="continue-wrapper"><span class="continue-hint">▼ 继续</span></div>
    </div>
    <div class="text-box" style="display:none;">
      <p class="narrative-text" data-fulltext="门上有三个奇怪的符号。">门上有三个奇怪的符号。</p>
      <div class="continue-wrapper"><span class="continue-hint">▼ 继续</span></div>
    </div>
  </div>

  <div class="interaction-panel" data-trigger="2" data-control-type="choice" data-config='{"options":["推开","观察","离开"]}'>
  </div>

  <script src="id:controls_library_v1"></script>
</div>
```

### 2.2 CSS样式师（Style Designer）

**职责**：根据场景氛围（可选）生成独立的样式代码，可以是内联 `<style>` 标签，也可以是纯 CSS 代码块。样式必须：

- **使用约定的类名**：与 JS 控件库中使用的类名保持一致（例如 `.choice-btn`、`.dice-btn`、`.dialogue-input` 等），确保动态生成的控件获得正确样式。
- **作用域**：建议通过根容器选择器（如 `#scene-{{timestamp}}`）限定样式范围，避免污染全局。
- **覆盖所有控件类型**：即使当前场景未使用某些控件，也应定义其样式，以备后续扩展。
- **设计自由**：颜色、布局、动画等完全由开发者根据剧情自由设计。

**示例样式**：
```css
<style>
#scene-1680000000000 {
  background-color: #2a1e1e;
  color: #f0e6d0;
  font-family: 'Times New Roman', serif;
}
#scene-1680000000000 .text-box {
  margin-bottom: 1.5em;
  line-height: 1.8;
}
#scene-1680000000000 .continue-hint {
  display: inline-block;
  margin-top: 0.5em;
  padding: 0.3em 1em;
  background: #8b5a2b;
  color: #fff;
  border-radius: 20px;
  cursor: pointer;
  transition: background 0.2s;
}
#scene-1680000000000 .continue-hint:hover {
  background: #b87c4b;
}
#scene-1680000000000 .choice-btn {
  display: block;
  width: 100%;
  padding: 10px;
  margin: 5px 0;
  background: #3d2b1a;
  border: 2px solid #8b5a2b;
  color: #f0e6d0;
  font-size: 16px;
  border-radius: 8px;
  cursor: pointer;
}
#scene-1680000000000 .choice-btn:hover {
  background: #5a3e28;
}
</style>
```

### 2.3 JS控件库（Control Library）

**职责**：预置的 JavaScript 模块，在 DOM 加载完成后自动扫描页面，查找所有 `[data-control-type]` 容器，根据类型创建交互控件并绑定事件。

**规范**：

- **自执行**：控件库应自执行，无需外部调用。
- **扫描容器**：遍历所有 `[data-control-type]` 元素，解析 `data-config` 属性。
- **渲染函数**：为每种控件类型定义渲染函数，接收容器 DOM 元素和解析后的配置对象，动态创建 UI 元素（**必须使用约定的类名**），并绑定事件。
- **结果返回**：用户交互后，调用全局回调函数 `window.__interactionResolver(result)`，其中 `result` 为字符串，格式由开发者定义（如 `"[选择: 推开]"`）。
- **依赖约定**：
  - 全局回调由互动师注入，控件库不应自行定义。
  - 不注入任何样式，完全依赖 CSS 样式师。
  - 可使用自定义事件、动画、音效等增强体验，但需确保与样式解耦。

**示例（伪代码）**：
```javascript
(function() {
  document.addEventListener('DOMContentLoaded', function() {
    document.querySelectorAll('[data-control-type]').forEach(container => {
      const type = container.dataset.controlType;
      const config = JSON.parse(container.dataset.config || '{}');
      if (type === 'choice') {
        renderChoice(container, config);
      } else if (type === 'dice') {
        renderDice(container, config);
      }
    });
  });

  function renderChoice(container, config) {
    config.options.forEach(opt => {
      const btn = document.createElement('button');
      btn.className = 'choice-btn';
      btn.textContent = opt;
      btn.addEventListener('click', () => {
        window.__interactionResolver?.(`[选择: ${opt}]`);
      });
      container.appendChild(btn);
    });
  }
})();
```

### 2.4 互动师（Interaction Renderer）

互动师是系统内置组件，行为固定，无需开发者实现。其职责包括：

- **拼接**：将 CSS 样式代码和 HTML 骨架按顺序合并（样式在前，HTML 在后）。
- **资源解析**：将 HTML 中所有 `src="id:xxx"` 和 `href="id:xxx"` 替换为从对应存储（如 ImageStore、AudioStore）加载的真实 URL（如 Blob URL）。
- **渲染**：将最终 HTML 插入交互容器（如模态框），并确保全局回调函数 `window.__interactionResolver` 已定义。
- **结果传递**：当控件库调用回调时，将结果原样返回给工作流。

---

## 3. 组件协作流程

```
场景描述
    ├─> HTML框架师 ──> 纯HTML片段
    └─> CSS样式师 ───> 样式代码块
                         ↓
                     互动师拼接
                         ↓
              [样式] + [HTML] + [控件库引用]
                         ↓
                 解析资源引用为真实URL
                         ↓
                     渲染交互界面
                         ↓
                 用户交互 → 控件库回调 → 返回结果
```

- **并行生成**：HTML框架师和CSS样式师可同时工作，互不依赖。
- **资源加载**：所有外部资源（图片、音频、JS库）均通过 `id:` 协议引用，由互动师在渲染前解析。
- **交互回调**：控件库通过全局回调函数将用户操作结果返回给互动师，互动师将其传递给工作流。

---

## 4. 接口规范

为确保各组件无缝协作，必须遵守以下接口约定。

### 4.1 HTML框架师输出规范

- **根容器**：必须有一个唯一 ID，例如 `id="scene-{{timestamp}}"`。
- **控件容器**：每个交互控件需包含：
  - `data-control-type`：控件类型标识（如 `"choice"`）。
  - `data-config`：JSON 字符串，包含控件配置。
- **资源引用**：使用 `src="id:xxx"` 或 `href="id:xxx"`。
- **库引用**：必须包含 `<script src="id:控件库ID">`。

### 4.2 CSS样式师输出规范

- **格式**：可以是内联 `<style>` 标签或纯 CSS 代码块。
- **作用域**：建议通过根容器 ID 限定样式，如 `#scene-{{timestamp}} .class`。
- **类名**：必须与 JS 控件库中使用的类名一致（例如 `.choice-btn`）。

### 4.3 JS控件库规范

- **自执行**：在 DOM 加载完成后自动扫描控件容器。
- **回调**：用户交互后调用 `window.__interactionResolver(result)`。
- **不依赖样式**：只负责创建元素并添加类名，不设置任何内联样式。

### 4.4 互动师规范

- **全局回调**：确保 `window.__interactionResolver` 存在。
- **资源解析**：将所有 `src="id:xxx"` 替换为可访问的 URL。

---

## 5. 扩展性设计

### 5.1 新增控件类型

1. **定义控件标识**：确定新的 `data-control-type` 值（如 `"slider"`）。
2. **扩展控件库**：在 JS 控件库中添加对应类型的渲染函数，实现 UI 创建和事件绑定。
3. **更新样式规范**：在 CSS 样式师的指导中增加新控件所需的类名（如 `.slider`、`.slider-value`）。
4. **更新 HTML框架师规范**：在 HTML 框架师的指导中增加新控件类型的配置格式示例，使其能生成正确的容器。

### 5.2 修改控件样式

仅需调整 CSS 样式师的提示词，重新生成样式代码。JS 控件库和 HTML 框架师完全不受影响。

### 5.3 替换/升级控件库

上传新版本的 JS 文件到系统存储，使用相同 ID 覆盖，或分配新 ID 并相应修改 HTML 框架师中的引用。所有引用该 ID 的场景将自动使用新版控件库。

### 5.4 支持多主题/多版本

可为不同视觉主题分配不同的 CSS 样式师，或为不同交互版本分配不同的控件库 ID，由 HTML 框架师根据上下文（如从概览中获取版本信息）选择引用。

---

## 6. 总结

本方案通过 HTML、CSS、JS 和互动师的分离设计，实现了：

- **生成效率**：HTML 和 CSS 可并行生成，JS 预置，大幅缩短总耗时。
- **视觉灵活性**：每个章节可拥有独特的 CSS 样式，完美匹配剧情氛围。
- **交互可靠性**：JS 控件库经过充分测试，体验一致，可独立升级。
- **系统可维护性**：各组件职责清晰，修改样式或交互互不干扰。

开发者可以基于此核心思路，自由定义控件类型、样式体系、交互逻辑，只需遵循接口规范即可构建高效、灵活、稳定的互动小说自动化创作系统。

# 模态框适配指南

本文档详细说明自动化小说创作系统中所有模态框（交互框、Markdown预览框、章节查看框等）的统一规格，以及排版师（AI角色）输出 HTML 时必须遵守的适配规范。确保您的 HTML 能够无缝嵌入系统模态框，正确执行、反馈并适配所有设备。

---

## 1. 模态框统一结构

所有模态框共享同一套底层布局，排版师只需关注内容区（Body）即可，无需区分具体类型。

```
+---------------------------------------+
| 标题区域（高度≈60px，不同框标题不同）   |
+---------------------------------------+
|                                       |
| 内容区 (Body)                          |
| 放置排版师HTML，可滚动                  |
| 内边距: 12px                            |
| 背景色: #0f172a                        |
|                                       |
+---------------------------------------+
| 底部按钮区 (高度≈60px，系统按钮)         |
| 样式：金色渐变，圆角8px                  |
+---------------------------------------+
```

### 1.1 标题区域
- 由系统动态填充，排版师无需处理。

### 1.2 内容区（Body）规格
- **最大宽度**：`max-width: 700px`，宽度自适应，居中显示。
- **背景色**：`#0f172a`（深色），文字颜色默认 `#eaeaea`，排版师可自由覆盖。
- **内边距**：四周 `12px`（已在 `.nc-modal-body` 中定义）。
- **滚动**：`overflow-y: auto`，内容超出时自动滚动。
- **图片支持**：所有 `<img src="id:xxx">` 标签会被系统自动替换为真实图片的 Blob URL。
- **脚本执行**：HTML 内的 `<script>` 标签会被执行，可用于交互逻辑。

### 1.3 底部按钮区
- **所有模态框底部均有固定按钮**：
  - **交互框**：两个按钮“复制内容”和“跳过”，样式为金色渐变。
  - **其他框**：通常为“复制”和“关闭”按钮，但样式一致。
- **高度**：约 60px（包含按钮内边距）。
- **排版师无需考虑此区域**：您的 HTML 只放置在内容区，不会与底部按钮重叠（内容区已预留滚动空间）。若需设计固定底部的交互面板，请确保其 `bottom` 值大于底部按钮高度（至少 `70px`），避免遮挡。

---

## 2. 交互框专用回调与延迟机制

仅交互框（用于互动小说）具有以下机制，其他模态框（如预览框）无需处理。

- **回调函数**：交互框全局暴露 `window.__interactionResolver(result)`，排版师必须在用户交互后调用它，传递用户操作结果（推荐 JSON 格式）。
- **延迟关闭**：调用回调后，系统自动禁用所有按钮、显示倒计时（5秒）、关闭模态框并返回结果。排版师无需自行处理关闭。

---

## 3. 排版师输出 HTML 的核心原则

排版师（`typesetter`）的职责：根据场景师输出的【画面】、【正文】、【交互描述】以及融合生图师生成的融合图 ID，合成为一个完整的初始 HTML 场景。输出的 HTML 必须遵循以下核心原则，以保证与系统的兼容性。

### 3.1 输入解析示例

**输入格式示例**：
```
【画面】
（画面描述，可能包含融合图ID）

【正文】
第一段文本

第二段文本

第三段文本

【交互描述】
触发时机: 第2段后
类型: choice
选项:
- 调查书桌
- 检查床铺
- 离开房间
```

**解析步骤**：
1. **提取融合图ID**：从【画面】中查找 JSON 格式 `{"fusion_image_id": "id:xxx"}` 或直接提取 `id:xxx` 部分。
2. **分割正文段落**：使用 `\n\n`（双换行符）分割【正文】部分，每段生成一个 `.text-box` 元素。
3. **解析交互描述**：
   - `触发时机`：提取关键词（“第N段后”或“结束”）。
   - `类型`：`choice`、`dice`、`dialogue`、`timer`。
   - `选项`：每行以 `-` 开头的内容。
   - `范围`：仅 dice 类型，格式如 `1-6`。
   - `提示`：仅 dialogue 类型，作为输入框 placeholder。
   - `时限`：仅 timer 类型，数字（秒）。

### 3.2 HTML 基本结构要求

所有内容必须包裹在一个具有**唯一 ID** 的容器内，推荐使用时间戳或随机字符串生成 ID。

```html
<div id="scene-{{timestamp}}" class="scene-container">
  <!-- 背景层（绝对定位） -->
  <div class="bg-layer">
    <img src="id:fusion_123" class="bg-image" alt="scene">
    <div class="vignette"></div>
  </div>

  <!-- 内容包装器：采用Flex列布局，包含正文和交互面板 -->
  <div class="content-wrapper">
    <!-- 正文层（可滚动） -->
    <div class="text-layer">
      {{paragraphs_html}}
    </div>

    <!-- 交互面板（始终位于正文下方） -->
    <div class="interaction-panel" data-trigger="{{trigger_point}}">
      {{interaction_controls}}
    </div>
  </div>

  <!-- 隐藏按钮（绝对定位） -->
  <button class="toggle-btn" aria-label="切换显示">👁</button>
</div>
```

- **为什么需要唯一 ID？**  
  脚本通过 `document.getElementById()` 准确获取自身容器，避免使用 `document.currentScript.previousElementSibling`（该方式在动态插入脚本时会失效）。
- **容器类名**：使用 `.scene-container` 作为命名空间，便于样式隔离。
- **重要原则**：`.scene-container` **不应设置 `transform` 属性**，以免成为内部绝对定位元素的包含块，导致固定定位失效。GPU 加速应仅应用于需要动画的独立元素（如 `.bg-layer`、`.text-box` 等）。

### 3.3 图片处理规范

- **插入图片**：使用 `<img src="id:xxx">` 格式，系统会自动替换为 Blob URL。
  ```html
  <div class="bg-layer">
    <img src="id:fusion_123" class="bg-image" alt="scene">
    <div class="vignette"></div>
  </div>
  ```

- **无图后备**：若输入中没有提供融合图 ID，应使用 CSS 渐变背景作为后备。
  ```css
  .bg-layer {
    background: linear-gradient(135deg, #1a1a2e 0%, #16213e 100%);
  }
  ```

- **暗角遮罩**：可添加 `vignette` 元素增强氛围，不影响图片显示。
  ```css
  .vignette {
    position: absolute;
    inset: 0;
    background: radial-gradient(ellipse at center, transparent 30%, rgba(0,0,0,0.85) 100%);
    pointer-events: none;
    transform: translateZ(0);
  }
  ```

### 3.4 正文段落与打字机效果

#### 3.4.1 段落结构
为每个段落生成一个 `.text-box` 元素，第一段默认显示，其余隐藏。每段文本需存储在 `data-fulltext` 属性中。

```html
<div class="text-layer">
  <div class="text-box">
    <p class="narrative-text" data-fulltext="第一段完整文本">第一段完整文本</p>
    <div class="continue-wrapper"><span class="continue-hint">▼ 继续</span></div>
  </div>
  <div class="text-box" style="display:none;">
    <p class="narrative-text" data-fulltext="第二段完整文本">第二段完整文本</p>
    <div class="continue-wrapper"><span class="continue-hint">▼ 继续</span></div>
  </div>
  ...
</div>
```

#### 3.4.2 打字机效果（高性能版）
使用 `requestAnimationFrame` 实现流畅打字，避免 `setTimeout` 卡顿。

```javascript
function typeParagraph(element, text, onComplete) {
  if (!element || !text) return;
  let index = 0;
  const total = text.length;
  const baseSpeed = Math.max(16, Math.min(35, 1000 / (total * 0.5 + 10)));
  let lastTime = performance.now();

  element.classList.add('typing');
  element.textContent = '';

  function step(currentTime) {
    const elapsed = currentTime - lastTime;
    const chars = Math.max(1, Math.floor(elapsed / baseSpeed));
    if (index < total) {
      index = Math.min(index + chars, total);
      element.textContent = text.substring(0, index);
      lastTime = currentTime;
      requestAnimationFrame(step);
    } else {
      element.classList.remove('typing');
      element.classList.add('typing-complete');
      onComplete?.();
    }
  }
  requestAnimationFrame(step);
}
```

#### 3.4.3 段落切换逻辑
- 点击“继续”按钮时，若当前段落在打字中，则立即完成打字；否则显示下一段。
- 每显示一段新段落，启动其打字机效果，完成后显示该段的“继续”按钮。
- 所有段落显示完毕后，检查交互触发时机。

### 3.6 回调机制与结果提交

用户与控件交互后，排版师必须调用全局函数 `window.__interactionResolver(result)` 将结果返回给系统。

#### 3.6.1 结果对象格式
推荐使用 JSON 格式，至少包含：
- `type`: 控件类型（`choice`/`dice`/`dialogue`/`timer`）
- `value`: 用户选择的值或输入内容
- `timestamp`: `Date.now()`
- `sceneId`: 当前场景容器的 ID（便于调试）

示例：
```javascript
{
  type: "choice",
  value: "调查书桌",
  timestamp: 1680000000000,
  sceneId: "scene-123456"
}
```

#### 3.6.2 调用时机
- **Choice**：点击按钮后立即调用。
- **Dice**：显示骰子结果动画后延迟调用（例如600ms）。
- **Dialogue**：点击发送按钮后调用。
- **Timer**：倒计时结束后调用（或用户主动停止时）。

#### 3.6.3 注意事项
- 调用回调后，系统会自动禁用所有交互控件、显示倒计时并关闭模态框，排版师**无需自行关闭**。
- 如果 `window.__interactionResolver` 不存在，应显示友好错误提示，避免脚本崩溃。
  ```javascript
  if (typeof window.__interactionResolver === 'function') {
    window.__interactionResolver(JSON.stringify(result));
  } else {
    showError('系统错误：无法返回结果');
  }
  ```

### 3.7 布局与底部空间预留（Flex列布局）

为确保交互面板始终位于正文下方且不随滚动滑动，同时避免与系统底部按钮重叠，推荐采用 **Flex列布局**。核心方案如下：

- 外层容器 `.scene-container` 设为 `display: flex; flex-direction: column; height: 100%;` 撑满模态框内容区。
- 背景层使用绝对定位，脱离文档流。
- 新增 `.content-wrapper` 作为相对定位的 Flex 容器，`flex: 1` 占据剩余高度，内部包含正文和交互面板。
- 正文区域 `.text-layer` 设置 `flex: 1; overflow-y: auto;` 使其可滚动。
- 交互面板 `.interaction-panel` 设置 `flex-shrink: 0;` 并添加适当外边距，使其始终位于正文下方。
- 隐藏按钮使用绝对定位，不受 Flex 影响。

**关键 CSS 示例**：
```css
.scene-container {
  position: relative;
  width: 100%;
  height: 100%; /* 撑满模态框内容区 */
  display: flex;
  flex-direction: column;
  background: var(--bg-dark);
  /* 注意：不要在此处设置 transform */
}

.bg-layer {
  position: absolute;
  inset: 0;
  z-index: 0;
  transform: translate3d(0, 0, 0); /* 背景可单独加速 */
}

.content-wrapper {
  position: relative;
  flex: 1;
  display: flex;
  flex-direction: column;
  overflow: hidden;
  z-index: 10;
  padding: 0 12px; /* 与模态框内边距对齐 */
}

.text-layer {
  flex: 1;
  overflow-y: auto;
  padding-top: 20px;
  padding-bottom: 20px;
  transform: translateZ(0); /* 可单独加速 */
}

.interaction-panel {
  flex-shrink: 0;
  margin: 0 0 20px 0;
  /* 其他样式 */
  opacity: 0;
  pointer-events: none;
  transition: opacity 0.3s ease;
}
.interaction-panel.active {
  opacity: 1;
  pointer-events: auto;
}
```

**为什么这样设计？**
- Flex 列布局天然保证交互面板位于正文下方，不会随滚动滑动。
- `flex: 1` 让正文区域自动占满剩余空间，当内容过多时滚动，面板始终可见。
- 底部外边距 `margin-bottom` 可微调面板与系统按钮的距离，系统按钮已在父容器外部，无需额外处理。
- 背景绝对定位不受 Flex 影响，可覆盖整个容器。

### 3.8 脚本健壮性与错误处理

#### 3.8.1 获取容器
**绝对禁止**使用 `document.currentScript.previousElementSibling`，应通过唯一 ID 获取：

```javascript
const container = document.getElementById('scene-{{timestamp}}');
if (!container) {
  console.error('[排版师] 找不到场景容器');
  return;
}
```

#### 3.8.2 元素存在性检查
对所有关键元素（`.content-wrapper`、`.text-layer`、`.interaction-panel` 等）进行存在性检查，若缺失则动态创建或提供后备。

```javascript
// 确保 content-wrapper 存在
let contentWrapper = container.querySelector('.content-wrapper');
if (!contentWrapper) {
  contentWrapper = document.createElement('div');
  contentWrapper.className = 'content-wrapper';
  container.appendChild(contentWrapper);
}

let textLayer = container.querySelector('.text-layer');
if (!textLayer) {
  textLayer = document.createElement('div');
  textLayer.className = 'text-layer';
  contentWrapper.prepend(textLayer);
}

let paragraphs = container.querySelectorAll('.text-box');
if (paragraphs.length === 0) {
  // 创建默认段落
  const defaultBox = document.createElement('div');
  defaultBox.className = 'text-box';
  defaultBox.innerHTML = '<p class="narrative-text" data-fulltext="场景加载中...">场景加载中...</p><div class="continue-wrapper"><span class="continue-hint">▼ 继续</span></div>';
  textLayer.appendChild(defaultBox);
  paragraphs = container.querySelectorAll('.text-box');
}
```

#### 3.8.3 事件委托
将事件监听挂载在容器上，使用事件委托处理所有子元素事件，避免因动态插入丢失绑定。

```javascript
container.addEventListener('click', (e) => {
  if (e.target.closest('.continue-hint')) { ... }
  if (e.target.closest('.interaction-panel')) { ... }
  if (e.target.closest('.toggle-btn')) { ... }
}, { passive: true });
```

#### 3.8.4 错误处理清单
| 情况 | 处理方式 |
|------|----------|
| 无融合图 ID | 使用 CSS 渐变背景替代 |
| 正文为空 | 自动生成默认段落“场景加载中...” |
| 无交互描述 | 不生成交互面板，仅展示正文 |
| 回调函数不存在 | 显示错误提示，不崩溃 |
| 重复点击 | 点击后立即禁用按钮，防止多次回调 |
| 打字机卡顿 | 使用 RAF 并动态计算速度，长文本自动加速 |
| 内存泄漏 | 页面卸载时取消 RAF，移除事件监听 |

### 3.9 性能优化清单

- [ ] **GPU 加速**：对需要动画的元素（背景、文本框、按钮等）添加 `transform: translateZ(0)` 或 `translate3d(0,0,0)`，**注意不要在固定定位元素的祖先上使用**。
- [ ] **will-change**：对打字机光标、悬停效果等添加 `will-change: transform, opacity`。
- [ ] **contain**：对滚动区域使用 `contain: layout style paint` 隔离渲染。
- [ ] **requestAnimationFrame**：所有动画（打字机、骰子旋转）均使用 RAF。
- [ ] **passive 事件**：滚动监听添加 `{ passive: true }`。
- [ ] **避免强制同步布局**：批量读写 DOM 属性，使用 `requestAnimationFrame` 批量更新样式。
- [ ] **CSS 变量**：使用 CSS 自定义属性减少 JavaScript 样式计算。
- [ ] **防抖节流**：窗口 resize 使用防抖（debounce），滚动使用节流（throttle）。
