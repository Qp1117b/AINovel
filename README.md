## 1. 系统概述

本系统是一个运行在 SillyTavern 页面的油猴脚本（`auto.js`），通过调用酒馆助手 API 实现多 Agent 的自动调用、状态书管理、章节生成与保存。其核心特性包括：

- **多 Agent 协作**：将创作流程分解为多个 Agent，每个 Agent 对应一张 SillyTavern 角色卡，可自定义输入、模板和执行顺序。
- **工作流阶段**：按顺序执行的步骤，每个阶段可包含多个 Agent，支持串行或并行执行。
- **状态书与快照**：使用世界书存储动态状态，每章自动创建快照，支持废章回滚和历史追溯。
- **跨章记忆**：通过 `.last` 引用上一章输出，实现增量更新。
- **回流机制**：当 Agent 输出特定关键词时，可精准触发相关 Agent 重新执行，并附上反馈。
- **多媒体生成**：支持图像生成（文生图、融合图、变体）、音频生成（音乐、语音、编辑），并自动嵌入章节。
- **Galgame 制作器**：基于节点脚本驱动的互动剧情编辑器，支持变量、分支和独立打包。
- **可视化配置编辑器**：内置图形化界面，可拖拽配置工作流阶段、Agent 和 API，实时校验。

## 2. 环境准备与安装

### 2.1 必需软件

- **SillyTavern**：最新版本（≥ 1.12.0）
- **酒馆助手插件（JS-Slash-Runner）**：最新版本，需支持 `custom_api` 参数
- **浏览器**：Chrome / Edge / Firefox（建议使用最新版）
- **油猴扩展**：Tampermonkey 或 Violentmonkey
- **（可选）图像生成后端**：如 Stable Diffusion WebUI 或其他支持 OpenAI 格式的图像 API
- **（可选）音频生成后端**：如 ElevenLabs、Stable Audio 等

### 2.2 安装 SillyTavern

1. 前往 [SillyTavern 官方 GitHub](https://github.com/SillyTavern/SillyTavern) 下载最新版本。
2. 按照官方指南解压并运行（通常为 `start.bat` 或 `node server.js`）。
3. 确保 SillyTavern 能正常访问（默认地址 `http://localhost:8000`）。

### 2.3 安装酒馆助手插件

1. 在 SillyTavern 的插件目录（`plugins`）中，通过 Git 克隆或下载 [JS-Slash-Runner](https://github.com/SillyTavern/JS-Slash-Runner) 的最新版本。
2. 重启 SillyTavern，在插件管理页面确认酒馆助手已启用。
3. 验证酒馆助手 API 是否可用：打开浏览器控制台，输入 `typeof TavernHelper`，应返回 `"object"`。

### 2.4 安装油猴脚本

1. 在浏览器中安装 Tampermonkey 或 Violentmonkey 扩展。
2. 将 `auto.js` 脚本内容复制到扩展中，创建一个新脚本并保存。
3. 确保脚本的 `@match` 字段匹配您的 SillyTavern 访问地址（例如 `http://127.0.0.1:8000/`）。
4. 刷新 SillyTavern 页面，脚本会自动加载。页面右下角应出现一个浮动按钮 **📚 创作**。

### 2.5 验证安装

- 打开浏览器开发者工具（F12），查看控制台是否显示 `[自动化小说创作系统 v1.0] 基础初始化完成` 等日志。
- 点击浮动按钮，若弹出前置检测失败面板，说明环境尚缺配置（正常情况，后续我们将配置）。

## 3. 核心概念速览

| 概念 | 说明 |
|------|------|
| **Agent** | 对应 SillyTavern 中的角色卡，执行特定任务。分为特殊角色和普通 Agent。 |
| **工作流阶段** | 按顺序执行的步骤，每个阶段可包含多个 Agent，**必须为纯串行或纯并行**。 |
| **预选配置** | 通过树形菜单选择分类选项，系统自动组合推荐的 Agent 列表。 |
| **状态书** | 存储动态状态的世界书，命名格式 `状态书-N`，包含模板条目和状态条目。 |
| **设定书** | 存储静态世界观的世界书，如 `设定书.json`。 |
| **输入源** | Agent 的 `inputs` 数组中的元素，可以是 `"user"`、`"auto"`、`"before"`、阶段 ID、其他 Agent 键、`"键.last"`、文件源等。 |
| **回流** | 当 Agent 输出中包含预设关键词时，触发其依赖的 Agent 重新执行，并附上反馈。 |
| **废章** | 当状态更新失败或连续回流超限时，当前章节被标记为废章，系统自动回滚状态书，但保留章节内容供查看。 |
| **生图师** | 特殊角色（`imageGenerator`），负责生成图像提示词并调用图像 API，将图片嵌入章节。 |
| **Galgame 制作器** | 基于节点脚本驱动的互动编辑器，可创建非线性剧情，支持变量、分支和独立打包。 |

## 4. 获取配置文件

系统需要加载一个 JSON 格式的配置文件，定义所有 Agent、工作流阶段、分类选项等。您可以通过以下方式获取：

- **手动编写**：参考《配置文件编写指南》，根据需求编写。
- **借助大模型生成**：向大模型提供需求，生成初稿后手动调整。
- **使用示例配置文件**：可从官方仓库获取示例。

**注意**：配置文件必须符合校验规则，否则加载时会报错。建议使用内置的**可视化配置编辑器**进行编辑（详见第14节），可避免格式错误。

## 5. 准备角色卡

### 5.1 特殊角色卡

根据您的需求，可能需要以下特殊角色卡（每个角色的输出格式有特定要求）：

- `finalChapter`（最终文章师）
- `optimizer`（状态优化师）
- `updater`（状态维护师）
- `storySummarizer`（剧情概览师）
- `imageGenerator`（生图师）
- `fusionGenerator`（融合生图师）
- `imageVariator`（变化生图师）
- `imageLibrarian`（图片管理员）
- `musicGenerator`（音乐生成师）
- `voiceCloner`（语音克隆师）
- `audioEditor`（音频编辑师）
- `audioLibrarian`（音频管理员）
- `interactiveAgent`（互动师）
- `typesetter`（排版师）
- `saver`（最终保存器）

每个特殊卡应按照系统要求的格式编写输出，确保能被正确解析。可从官方社区获取示例卡或自行编写。

### 5.2 普通 Agent 角色卡

其他普通 Agent 可以复用同一张通用角色卡。该卡应设计为能够理解 `inputTemplate` 中指定的角色描述，并遵循相应的输出格式。通用卡的 `system_prompt` 中应包含：“用户会通过输入文本描述你当前应扮演的角色，你必须立即理解该角色并以该角色的身份思考和行动。”

您只需维护一张通用卡，所有职能差异通过配置文件中的 `inputTemplate` 实现。

**角色卡导入**：在 SillyTavern 中，将编写好的角色卡 JSON 文件拖入“导入角色”区域即可。

## 6. 准备世界书

### 6.1 设定书（静态世界观）

设定书（通常名为 `设定书.json`）存储静态世界观信息，如地理、势力、角色背景等。其条目应合理设置触发关键词、`position`、`depth` 等。核心条目可设为 `constant: true` 始终注入。

### 6.2 状态书（动态状态）

状态书用于存储动态状态，命名格式为 `状态书-N.json`（N 从 1 开始）。每本状态书必须包含：

- **一个模板条目**：名称 `状态模板-N`，内容定义所有类别及字段结构（不填具体值）。
- **多个状态条目**：名称 `状态-{书号}-{类别编号}-{类别名}`，内容初始与模板相同（占位符），后续由维护师更新。

**创建方式**：
- 手动编写：按上述格式创建 JSON 文件并导入 SillyTavern。
- 系统自动创建：首次启动时，若检测到状态书缺失，系统会尝试创建空的状态书（仅含模板条目，无状态条目），但建议您提前准备好。

## 7. 加载配置与启动创作

### 7.1 加载配置文件

1. 点击浮动按钮 **📚 创作**，打开主面板。
2. 若前置检测失败，面板会显示错误信息。此时点击 **📂 加载配置文件** 按钮，选择您准备好的 JSON 配置文件。
3. 系统会进行校验，若通过则自动重新打开面板，并显示加载成功。

### 7.2 选择预选模式

主面板左侧是树形预选菜单，您可以根据需要选择不同的分类选项（如质量分层、章节类型）。系统会根据选中的选项自动勾选右侧的 Agent 复选框（预选模式）。您也可以切换到“自定义模式”手动勾选 Agent。

**注意**：四个核心角色（如 `finalChapter`、`optimizer` 等）如果配置为 `required: true`，则始终被勾选且不可取消。

### 7.3 启动工作流

1. 在右侧文本框中输入本章的创作需求（如“主角遇到新的挑战，需要做出选择”）。
2. 确认自动执行复选框的状态（勾选后系统会连续生成多章，直到您中断）。
3. 点击 **▶️ 启动** 按钮，系统开始执行工作流。您会在“执行进度”区域看到实时日志，Agent 状态按钮也会变化。

**第一次运行**：由于跨章记忆为空，第一章可能会触发多次用户输入（如要求提供初始概览）。请根据提示在文本框中输入必要信息，并点击 **✍️ 提交输入**。一旦记忆建立，后续章节将自动运行。

## 8. 监控与干预

### 8.1 进度日志与状态按钮

- **进度日志**：显示每个 Agent 的开始、完成、错误、回流等信息，以及用户输入提示。
- **Agent 状态按钮**：在工作流预览区，每个 Agent 按钮的颜色表示其状态（具体颜色见下表）：

| 状态 | 颜色 | 说明 |
|------|------|------|
| `idle` | 淡紫色 | 空闲，等待执行 |
| `running` | 橙色 | 正在执行中 |
| `pending` | 紫色 | 等待依赖（回流等待） |
| `waiting_input` | 青蓝色渐变 | 等待用户输入 |
| `reflow_processing` | 粉红色渐变 | 回流处理中 |
| `reflow_waiting` | 浅红色 | 等待回流目标完成 |
| `completed` | 绿色渐变 | 执行完成 |
| `error` | 红色渐变 | 执行出错 |

点击按钮可查看该 Agent 本章的输出。

### 8.2 处理用户输入

当 Agent 需要用户输入时（例如依赖缺失、`.last` 首次为空），系统会：

- 在进度区域显示提示词（来自 `inputPrompts`）。
- 根据 `inputMode` 显示不同的提交按钮：
  - **✍️ 提交输入**：直接使用文本框内容。
  - **📚 章节状态**：打开章节选择模态框，可选择历史章节自动提取内容（支持文章、状态或两者）。
  - **📋 要求**：显示当前等待的具体要求（点击可查看详细提示）。
  - **📖 读取文件**：当输入源为 `read.xxx` 时，点击打开文件选择器，上传文件后自动提交。
  - **💾 保存文件**：当输入源为 `save.xxx` 时，点击打开文件选择器，上传文件后自动保存并返回 ID。

输入后点击对应按钮提交。系统会自动继续执行。

### 8.3 废章处理与回滚

如果状态维护师更新失败（例如协议解析出错），或者同一源连续触发回流超过 `maxConsecutiveReflows`，本章会被标记为废章：

- 系统自动回滚状态书到本章开始前的状态。
- 废章内容保存在内存中，可通过点击工作流预览区的**废章**按钮查看。
- 您可以选择重新启动本章（需修改输入），或忽略继续下一章。

## 9. 图像生成详解

### 9.1 图像 API 配置

在配置文件的 `apiConfigs` 中必须定义一个 `type: "image"` 的配置，且整个文件只能有一个。例如：

```json
"sdWebUI": {
    "type": "image",
    "mode": "txt2img",
    "source": "sdwebui",
    "apiUrl": "http://127.0.0.1:7860",
    "key": "",
    "model": "v1-5-pruned-emaonly.safetensors",
    "steps": 20,
    "cfg_scale": 7,
    "width": 512,
    "height": 512,
    "sampler_name": "Euler",
    "negative_prompt": "nsfw, low quality",
    "timeout": 300000
}
```

支持的 `source` 枚举：`openai`、`stability`、`midjourney`、`sdwebui`、`siliconflow` 等。必须选择正确的 `source` 值，否则图像生成将失败。

### 9.2 生图师输出格式

生图师（`role: "imageGenerator"`）的输出必须包含图片生成指令块，格式如下：

```
===图片生成指令===
段落1-2：
提示词内容（可跨多行）
段落3：
提示词内容
...
```

- 以 `===图片生成指令===` 为起始标记。
- 每个任务以 `段落<start>[-<end>]：` 开头，指定该图片对应的段落范围（从 1 开始计数）。
- 系统解析后，会为每个任务调用图像 API，并将生成的图片保存到 `ImageStore`，返回图片 ID。最终章节中会在对应段落之后插入图片占位符 `![图片](id:xxx)`。

### 9.3 图片存储与清理

- 所有生成的图片存储在浏览器的 IndexedDB 中（`ImageStore`）。
- 图片在以下情况会被自动清理：
  - **生图师回流时**：重新生成前会删除旧图片。
  - **删除章节时**：解析章节内容中的图片 ID 并删除对应图片。
  - **清空历史章节时**：调用 `ImageStore.clear()` 清空所有图片。
  - **一键纯净**：清空所有图片。
  - **废章处理**：废章产生的图片会被删除。
- 用户手动停止工作流时，也会清理已生成但未保存的图片。

## 10. 音频生成详解

### 10.1 音频 API 配置

音频 API 配置需指定 `type: "audio"` 和 `mode`（音乐生成、语音克隆、音频编辑）。例如：

```json
"elevenlabs": {
    "type": "audio",
    "mode": "voice-cloning",
    "source": "elevenlabs",
    "apiUrl": "https://api.elevenlabs.io/v1",
    "key": "your-api-key",
    "model": "eleven_multilingual_v2",
    "timeout": 60000
}
```

同一 `mode` 的音频配置只能有一个（如 `music-generation` 只能出现一次）。

### 10.2 各音频角色输出格式

- **音乐生成师**（`musicGenerator`）：
  ```json
  [
    { "name": "战斗音乐", "range": [1,2], "id": "audio_xxx" }
  ]
  ```
- **语音克隆师**（`voiceCloner`）：
  ```json
  [
    { "name": "角色语音", "sample_id": "audio_yyy", "text": "你好", "id": "audio_zzz" }
  ]
  ```
- **音频编辑师**（`audioEditor`）：
  ```json
  [
    { "name": "编辑后音频", "source_id": "audio_xxx", "id": "audio_yyy" }
  ]
  ```

### 10.3 音频存储与播放

- 音频文件存储在 `AudioStore`（IndexedDB）中。
- 在章节 HTML 中可通过 `<audio src="id:audio_xxx">` 引用，系统渲染时会自动替换为 Object URL。
- 清理规则同图片。

## 11. Galgame 制作器

本系统内置了功能强大的 **Galgame 制作器**，允许您基于已生成的章节构建非线性互动小说。制作器采用**节点脚本驱动**的设计：每个节点包含一段 JavaScript 脚本，可读写项目级变量，并根据用户交互结果返回目标节点 ID，从而实现动态剧情分支。

### 11.1 制作器界面与基本操作

在主面板底部点击 **🎮 Galgame制作器** 按钮，打开制作器。界面分为三个主要区域：

- **左侧资源库**：显示当前 `ImageStore` 和 `AudioStore` 中的所有图片和音频资源，方便您预览。
- **中央画布**：可视化编辑区域，您可以通过拖拽创建、连接节点。画布上显示淡蓝色的列辅助线，帮助您对齐节点。
- **右侧属性面板**：显示当前选中节点的详细属性，并可进行编辑。

#### 11.1.1 制作模式与播放模式

制作器顶部工具栏提供两种模式：

- **✏️ 制作模式**：默认模式，用于编辑节点、设置属性。
- **▶️ 播放模式**：切换到播放模式，您可以直接测试当前项目的流程，体验玩家视角。点击画布下方的“返回”按钮可回到制作模式。

#### 11.1.2 节点操作

- **创建节点**：在画布空白处双击，即可在鼠标位置创建一个新节点。新节点默认标题为“场景X”，引用章节号为 0（需后续关联）。
- **移动节点**：单击节点并拖拽可移动位置。按住 `Ctrl` 可多选，多选后拖拽可同时移动多个节点。拖拽时显示对齐线（绿色虚线）和列边界辅助线（淡蓝色虚线），帮助您对齐。
- **删除节点**：选中节点后按 `Delete` 键，或点击属性面板中的 **🗑️ 删除节点** 按钮。
- **复制/粘贴**：选中节点后按 `Ctrl+C` 复制，`Ctrl+V` 粘贴。粘贴的节点会自动放置于画布中心附近。
- **全选**：`Ctrl+A`。
- **撤销/重做**：`Ctrl+Z`（撤销），`Ctrl+Shift+Z` 或 `Ctrl+Y`（重做）。
- **自动布局**：点击工具栏上的 **🔄 自动布局** 按钮，系统会自动将当前所有节点按列对齐并垂直排列。

### 11.2 节点属性详解

当选中一个节点时，右侧属性面板显示以下可编辑字段：

| 字段 | 说明 |
|------|------|
| **节点ID** | 节点的唯一数字标识，系统自动分配，不可修改。 |
| **标题** | 节点的显示名称，在画布上显示。 |
| **引用章节号** | 该节点对应的历史章节号。播放时会加载此章节的内容。点击 **📖 从历史导入** 按钮可打开历史章节选择框，选中后自动填入章节号。 |
| **节点脚本 (script)** | JavaScript 代码，处理用户交互结果。脚本可访问 `vars` 和 `result`，应返回目标节点ID（数字）或路径字符串。若脚本无返回值或返回无效值，则使用“默认目标”。 |
| **默认目标 (defaultTarget)** | 当脚本未返回有效目标时使用的后备目标，可输入节点ID数字或路径字符串。 |
| **进入脚本 (onEnterScript)** | JavaScript 代码，在每次进入该节点时执行。可用于初始化变量、播放音效等。脚本可访问 `vars`。 |
| **变量监视器** | 显示当前项目所有变量的初始值，并可在制作时动态修改。您可以通过下方的 **➕ 新增变量** 按钮添加变量，并设置初始值。变量值支持数字、字符串、布尔值、数组、对象等 JSON 类型。 |
| **变量分析** | 自动分析当前节点的脚本，列出所有引用的变量及其可能的赋值情况，帮助您理解变量影响。 |

### 11.3 脚本执行环境

- `vars`：当前项目的全局变量对象，可读写。
- `result`：用户交互返回的字符串，格式通常为 `[类型: 值]`，例如：
  - 选项按钮点击 → `[选择: 接受委托]`
  - 滑块提交 → `[滑块: 42]`
  - 文本输入 → `[输入: 你好]`
- 内置工具函数 `utils`：
  - `utils.getControl(result)`：解析交互结果，返回 `{ type, value }`。
  - `utils.getNumber(result)`：提取数值（如滑块值）。
  - `utils.getText(result)`：提取文本（如选项文本）。

**示例脚本**：
```javascript
// 节点脚本：根据用户选择更新变量并返回目标ID
let choice = utils.getText(result);
if (choice === "接受委托") {
    vars.questAccepted = true;
    return 5;          // 跳转到节点5
} else if (choice === "拒绝") {
    vars.trust -= 10;
    return 6;
}
// 无匹配时，返回 undefined 以使用默认目标
```

### 11.4 变量管理

项目变量存储在 `WORKFLOW_STATE.galProject.variables` 中，可在整个游戏生命周期内读写。在播放模式中，变量是全局的，跨节点保持状态。在属性面板的“变量监视器”中，您可以：

- **查看当前所有变量及其初始值**。
- **修改变量值**：直接在表格的“初始值”输入框中修改（JSON 格式）。
- **删除变量**：点击变量行后的“✖”按钮。
- **新增变量**：点击 **➕ 新增变量** 按钮，输入变量名和初始值（JSON 格式）。

### 11.5 项目保存、加载与导出

制作器顶部工具栏提供以下按钮：

- **🆕 新建**：清空当前编辑，开始新项目。
- **💾 保存**：将当前项目（包括节点和变量）保存到 IndexedDB。若项目已有 ID，则覆盖；否则生成新 ID 并保存。
- **📂 加载**：打开项目列表，选择已保存的项目加载。
- **📤 导出**：将当前项目打包为 ZIP 文件，包含引用的章节、图片、音频、文本资源以及 `project.json`（含变量）。可用于分享或备份。
- **📥 导入**：从先前导出的 ZIP 文件恢复项目。系统会提示资源冲突（如需覆盖或跳过）。
- **📦 打包**：将当前项目打包为独立的 HTML 文件。该文件包含所有资源和变量初始值，可在任何浏览器中直接播放，无需依赖 SillyTavern 环境。

### 11.6 播放模式体验

在制作器中点击 **▶️ 播放模式**，画布变为播放界面：

- **中央内容区**：渲染当前节点引用的章节内容，图片、音频等资源自动嵌入。
- **底部交互区**：显示当前节点中由脚本生成的交互控件（如选项按钮、滑块、文本输入框等）。每个控件触发后会调用 `window.__interactionResolver` 将结果传递给播放器。
- **顶部工具栏**：显示当前节点标题，并提供 **返回** 按钮（可回退到上一个历史节点）。
- 播放过程中，变量变化实时生效，并影响后续分支。

### 11.7 注意事项

- **脚本必须返回数字或字符串**：若返回其他类型或无返回值，系统将使用 `defaultTarget`。建议在脚本末尾显式返回。
- **变量作用域**：变量在整个项目中全局共享，不同节点均可读写。谨慎修改变量名以避免冲突。
- **脚本调试**：播放模式下，浏览器控制台会输出脚本执行结果和错误信息，可利用 `console.log` 辅助调试。
- **资源清理**：播放模式关闭后，临时创建的图片、音频 URL 会自动释放，避免内存泄漏。
- **打包独立 HTML**：打包后的文件包含所有资源，体积可能较大，建议压缩图片音频后使用。

## 14. 配置编辑器使用指南

本系统内置了**可视化配置编辑器**，您可以通过图形界面轻松创建和修改配置文件，无需手动编写 JSON。编辑器提供以下功能：

### 14.1 打开编辑器

在主面板底部点击 **⚙️ 配置GUI** 按钮，打开配置编辑器。

### 14.2 界面布局

编辑器分为三个区域：

- **左侧资源面板**：列出所有 Agent、API 配置、分类和互斥组。点击可选中对应元素，在右侧属性面板编辑。
- **中央画布**：显示工作流阶段的节点和连接线。您可以拖拽阶段节点调整顺序，在阶段节点之间拖拽 Agent 进行移动。
- **右侧属性面板**：根据当前选中对象（阶段、Agent、API、分类、组）动态显示可编辑字段。

### 14.3 工作流阶段编辑

- **添加阶段**：在资源面板的“阶段管理”区域点击 **➕** 按钮，输入阶段 ID 和名称。
- **删除阶段**：选中阶段节点后按 `Delete` 键，或点击属性面板的“删除阶段”按钮。删除阶段时，阶段内的 Agent 会自动移至左侧未分配区，不会丢失。
- **调整顺序**：直接拖拽阶段节点即可改变执行顺序，连线会自动更新。
- **修改模式**：在属性面板中，可通过单选按钮将阶段切换为“串行”或“并行”。

### 14.4 Agent 编辑

- **添加 Agent**：在资源面板的“Agent 管理”区域点击 **➕**，输入唯一键名。新 Agent 会出现在左侧未分配区，您可将其拖拽到任意阶段。
- **移动 Agent**：
  - **跨阶段移动**：按住 Agent 拖拽到目标阶段节点上，即可将其移动到该阶段。
  - **拖入未分配区**：将 Agent 拖拽到画布空白处，即可将其从阶段中移除（移入未分配区）。
- **修改所属阶段**：通过“stage”下拉框可重新分配阶段。
- **输入源列表**：在属性面板中，您可以自由添加、删除、编辑输入源。候选列表会根据当前阶段自动过滤（只允许引用已执行过的 Agent 和阶段），确保依赖关系的正确性。

### 14.5 API 配置编辑

- **添加 API**：在资源面板的“API 配置”区域点击 **➕**，输入唯一 ID，选择类型（text/image/audio）。
- **类型特定字段**：根据选择的类型，属性面板会显示对应的参数（如图像尺寸、采样器、音频模式等）。系统会限制同一 `mode` 的音频配置只能有一个。
- **测试连通性**：在属性面板底部点击“测试连通性”按钮，可实时验证 API 是否可用。

### 14.6 分类与互斥组编辑

- **分类**：添加分类后，可设置选择模式（单选/多选），并为每个选项关联一组 Agent。
- **互斥组**：定义组内分类互斥，即组内只能选中一个分类的选项。

### 14.7 实时校验与导出

- **检测配置**：点击工具栏的 **🔍 检测配置** 按钮，可对当前配置进行完整校验，错误会显示在右侧面板。
- **应用配置**：校验通过后，点击 **✅ 应用配置** 可立即加载到主系统，无需重启。
- **导出 JSON**：点击 **📤 导出JSON** 可将当前配置保存为文件。
- **导入 JSON**：点击 **📥 导入JSON** 可从文件加载配置到编辑器。

## 15. 异常处理与常见问题

### 15.1 前置检测失败

- **症状**：启动时弹出错误面板，列出缺失的 Agent、状态书或 API 不可用。
- **解决**：
  - 确保所有 Agent 名称与角色卡完全一致。
  - 导入缺失的状态书模板或让系统自动创建。
  - 检查 API 配置的 URL、密钥和模型是否正确。

### 15.2 配置文件加载失败

- **症状**：加载后提示“配置文件校验失败”并列出错误。
- **常见错误**：
  - **输入源无效**：确保 `.last` 后缀使用 Agent 键，阶段 ID 已定义。
  - **占位符数量不匹配**：`inputTemplate` 中的 `【】` 数量必须等于 `inputs` 长度。
  - **角色重复**：特殊角色只能出现一次。

### 15.3 Agent 调用超时或无响应

- **解决**：
  - 检查网络连接和 API 服务是否正常。
  - 在 `apiConfigs` 中增加 `timeout` 值。
  - 手动测试对应角色卡是否能正常响应。

### 15.4 图像生成失败

- **常见原因**：
  - 图像 API 配置错误（source 不对、采样器名称无效）。
  - 生图师输出的段落范围超出实际段落数。
  - 图片保存失败（检查 IndexedDB 权限和存储空间）。

### 15.5 回流死循环

- **解决**：
  - 检查 `reflowConditions` 中的关键词是否过于宽泛。
  - 调整 `maxConsecutiveReflows` 和 `maxReflowDepth` 配置。
  - 优化 Agent 提示词，使其在接收到反馈后能生成不再触发回流的输出。
